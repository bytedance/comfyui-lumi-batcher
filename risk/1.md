根因
lumi_batcher_service/common/file_path.py 的 is_under_delete_white_dir() 不是判断"目标路径是否包含在允许的基目录内",
而是判断"resolve 后的路径里是否存在某个 名为 input/output/comfyui-lumi-batcher/comfyui_lumi_batcher_workspace 的组件":

white_folder = white_str.rstrip("/").split("/")[-1] # 取白名单目录的最后一段名字
path_components = [c for c in path_str.rstrip("/").split("/") if c]
if white_folder in path_components: # 只要任意一段同名就放行
return True
因此 <cwd>/comfyui_lumi_batcher_workspace/upload/../../custom_nodes/comfyui-lumi-batcher/prestartup_script.py
resolve 后仍含有 comfyui-lumi-batcher 组件 → 放行。上传目录恒为 <cwd>/comfyui_lumi_batcher_workspace/upload，
插件目录恒为 <cwd>/custom_nodes/comfyui-lumi-batcher，../../ 偏移与部署绝对路径无关，可靠且通用。
